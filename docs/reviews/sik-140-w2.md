## Scope

## Round 1 Findings

1. `high` `CalendarPeekProvider.tsx` / `CalendarPeekCard.tsx`
   - 证据：provider `state.list` 仍是 `open()` 时快照；card 成功保存只写本地 `committedPatch`。
   - 问题：同一轮 peek 会话里 `next/prev` 后再回到原事件会回旧值。
   - 处理：新增 `commitEvent(eventId, patch)` 回写 provider snapshot；移除 card 本地 `committedPatch`，成功保存后直接 `peek.commitEvent(...)`。

2. `medium` `apps/web/src/mocks/handlers/home.ts`
   - 证据：模块级 `cachedEvents` / `cachedEventsStamp` 持久缓存，并在 PATCH 中原地改写。
   - 问题：成功 smoke 后会污染同一天后续 GET 基线，破坏复跑确定性。
   - 处理：改为 5 秒 transient override，仅覆盖当前成功 refetch 窗口，随后自动回落到 `defaultEvents()` 基线。

3. `low` `CalendarPeekCard.test.tsx`
   - 证据：原测试只覆盖单事件 save/rollback，没有覆盖 round-trip 和同 event 既有 optimistic patch 保留。
   - 问题：对 provider snapshot stale / unrelated optimistic patch cleanup 缺少回归保护。
   - 处理：新增
     - `keeps a saved prop value after stepping away and back inside the same peek session`
     - `preserves unrelated optimistic patches for the same event after a successful save`
