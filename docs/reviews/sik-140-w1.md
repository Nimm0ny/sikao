## Findings

### Initial Findings (Resolved Before Pass)

1. `F1` `High` — successful save reverted to stale snapshot value because Peek read from provider list snapshot after optimistic patch removal.
   Resolution:
   - `CalendarPeekCard.tsx` now preserves a local committed patch per event id and restores prior optimistic patches instead of dropping the entire entry.

2. `F2` `Medium` — W1 could remove unrelated optimistic fields on the same event.
   Resolution:
   - success/failure now restore the pre-existing optimistic entry instead of blindly deleting the whole event patch.

3. `F3` `High` — sibling edit triggers stayed active, allowing silent draft loss when switching fields.
   Resolution:
   - title/notes `Edit` triggers now disable while any field is editing or saving.

4. `F4` `Medium` — title `Enter` save did not guard IME composition.
   Resolution:
   - title and notes save keyboard paths now skip `nativeEvent.isComposing`.

5. `F5` `Medium` — editing state still allowed head prev/next to discard drafts by navigating away.
   Resolution:
   - head prev/next now disable during editing; saving continues to disable prev/next/close.
