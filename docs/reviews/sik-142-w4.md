## Findings

### Initial Findings (Resolved Before Pass)

1. `F1` `High` — legacy `today` normalization only ran once on first mount, so delayed hydration could be silently ignored.
   Resolution:
   - `Home.tsx` now calls `hydrateFromLocalFallback()` on mount.
   - View sync now reacts to `preferences` / `profileLoaded` changes rather than reading the store only once.

2. `F2` `Medium` — earlier tests/browser evidence could not prove whether `today -> week` was a real normalization or just the store default.
   Resolution:
   - `Home.test.tsx` now covers:
     - legacy `today` normalization from a non-default `month` starting state
     - delayed hydration after mount
     - local fallback persisted `month`
   - Browser smoke now proves both:
     - persisted `month` selects `本月`
     - legacy `today` selects `本周`
