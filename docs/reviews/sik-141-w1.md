## Round 1 Findings

1. `high`
   - 问题：`mixed mock -> unsupported_track` 在实现层没有真正成立，含 subjective 子项的 session 仍可能误进 `ready`。
   - 处理：批量加载 answer + question facts，检测 `answer_kind`，只要混入非 objective 题型就返回 `unsupported_track`。

2. `medium`
   - 问题：前端 `normalizeAggregateEventIds()` 静默 `trim + filter` 空白 ID，吞掉了本应由后端 422 fail-fast 暴露的非法输入。
   - 处理：收口成只做 `Set + stable sort`，不再静默过滤。

3. `medium`
   - 问题：event/session 解析层仍然是 per-event N+1。
   - 处理：改成批量加载 concrete events、recurring parents、reverse links、occurrence links、sessions、answer facts；虚拟 occurrence 只保留内存级 rule 解析。

4. `medium`
   - 问题：Postgres 测试里的 `session_not_found` fixture 原先想靠悬空 FK 伪造，在真 Postgres 下不成立。
   - 处理：改成“事件指向当前用户不可见但结构合法的 foreign session”，从而在 FK 有效前提下覆盖 `session_not_found`。

## Round 2 Findings

- 无 findings。
