# SIK-140 W3 Review

## Scope

- `apps/web/src/components/form/Select/{Select.tsx,Select.test.tsx}`
- `apps/web/src/views/Home/sections/WeekCalendarView.test.tsx`
- `apps/web/src/views/Home/sections/peek/{CalendarPeekCard.tsx,CalendarPeekCard.test.tsx,CalendarPeekCard.module.css,CalendarPeekNotes.tsx,CalendarPeekProperties.tsx}`

## Round 1 Findings

1. `medium` `CalendarPeekCard.tsx` / `CalendarPeekProperties.tsx` / `Select.tsx`
   - 证据：W3 首版把 `Enter` 固定成 props editor 的保存快捷键，但 `Select` trigger 本身还不支持键盘用户在关闭态有效改值。
   - 问题：focus 虽然落到了 combobox，上下键并不能独立完成字段修改，真实变更路径仍依赖鼠标。
   - 处理：`Select` 新增 `enabledOptions/nextOptionByDirection`；非 searchable trigger 在关闭态支持 `ArrowUp/ArrowDown` 直接切换当前值，`Enter/Space` 保留 trigger 开关语义，props editor 再通过 passthrough 的 `Enter` 走保存分支。

2. `low` `CalendarPeekCard.test.tsx` / `Select.test.tsx`
   - 证据：首版新增用例仍是“鼠标选 option 后再 `fireEvent.keyDown(Enter)`”。
   - 问题：不能证明 `Enter save` 与 `Select` 自己的状态机在真实键盘路径下不冲突。
   - 处理：测试重写成 UTF-8，并把关键用例改成 keyboard-only `ArrowDown + Enter`；`Select.test.tsx` 也新增关闭态箭头切换值覆盖。
