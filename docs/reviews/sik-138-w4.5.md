# Findings

1. `High` Historical correctness bug: the W4.5 snapshot still let `TodayCalendarView` misplace recurring cross-day carry-in/carry-out geometry.
   Evidence anchors:
   - `packages/calendar-engine/src/recurrence.ts@52c09fef9:68-71`
   - `apps/web/src/views/Home/sections/TodayCalendarView.tsx@52c09fef9:50-59`
   - `apps/web/src/views/Home/sections/TodayCalendarView.tsx@52c09fef9:117-119`
   Evidence:
   - `expandRecurringOccurrences(...)` intentionally returns occurrences that overlap the window, including carry-in from before the anchor day.
   - Historical `TodayCalendarView.tsx` computed geometry directly from `occurrence.startAt/endAt` without clipping to the active day window.
   Impact:
   - A recurring event spanning midnight could render at the wrong horizontal position/width in the historical today timeline.
   Resolution status on current `main`:
   - Superseded by `SIK-142` W4: `TodayCalendarView` has been removed from Home entirely, so this historical bug no longer has a live surface.

2. `Medium` Projection-aware empty-state gating was incomplete: month/week historically keyed empty/ready UI off raw `events.length` rather than projected `occurrences/slices`.
   Evidence anchors:
   - `apps/web/src/views/Home/sections/TodayCalendarView.tsx@52c09fef9:121,140-156`
   - `apps/web/src/views/Home/sections/WeekCalendarView.tsx@52c09fef9:173-178,197-203`
   - `apps/web/src/views/Home/sections/MonthCalendarView.tsx@52c09fef9:164-169,188-199`
   - `packages/calendar-engine/src/recurrence.ts@52c09fef9:76-100`
   - `packages/calendar-engine/src/view.ts@52c09fef9:80-106`
   Evidence:
   - Historical month/week code checked raw query event count.
   - If a recurring parent was fully filtered out by `recurringExceptionDates`, the UI could land in a blank ready state instead of a proper empty state.
   Resolution status on current `main`:
   - Fixed in this run by switching:
     - `MonthCalendarView.tsx` to `visibleSliceCount`
     - `WeekCalendarView.tsx` to `visibleOccurrenceCount`
   - Added regression tests for “query returns one recurring parent, projection returns zero visible rows” in both month and week.

3. `Medium` Historical W4.5 tests proved helper projection but under-covered view-level seams.
   Evidence anchors:
   - `apps/web/src/views/Home/sections/calendarEvents.test.ts@b0a865037:68-163`
   - `apps/web/src/views/Home/sections/MonthCalendarView.test.tsx@b0a865037:115-129`
   - `apps/web/src/views/Home/sections/TodayCalendarView.test.tsx@b0a865037:58-96`
   Evidence:
   - `calendarEvents.test.ts` validated the projection helpers well.
   - Historical month/today tests did not fully lock multi-surface behavior for carry-in/carry-out and zero-projection UI.
   Resolution status on current `main`:
   - Partially reduced in this run by adding zero-projection empty-state tests for month/week.
   - Remaining residual gap: no dedicated browser/integration coverage exists for historical today carry-in behavior because the today view has since been removed.
