申论考场 V3 · 思考版

从头来。主调真考紧迫（真倒计时 / 不打扰主流程），UI 上集中发力 4 处差异化（真稿纸 / 智能草稿 / 翻页 / 资料"纸"质感），平板和键盘双适配。划线只划线（取消 tag），分类挪到草稿卡环节。

主调

真考紧迫为主。真倒计时（不温和）、答题主流程不打弹窗。UI 上发力差异化（A 优先），评分内容后补。平板用户偏多但电脑键盘 type 也是亮眼方向，双适配。

4 个核心 wow

差异化集中在这 4 处，其它都是 supporting。

1视觉锤
真 25×20 稿纸 + 翻页动画

键盘 type 字进格，写满 fold-page 翻页。500 字 / 页，1000 字大作文 = 2 页。视觉直接喊出"申论 = 写"。

2智能草稿
草稿 = 卡片墙 + 自动出处

拖材料文字进草稿区 → 自动落卡 with 资料 N · §M。自由打字也能成卡。整理时给卡选 论 / 据 / 数 / 策 分类。

3纸质感
资料区"纸"质 + 翻书页码

暖白底（#faf8f1）+ 米色格线 + 真翻页（动画 600ms 3D rotateY）。强化考场感，区别于"网页阅读器"。

4双手感
平板触摸 + 键盘 hot keys

平板：swipe 切资料 / 双指上滑翻页 / 长按拖卡片。键盘：⌘← 前页、⌘→ 下页、⌘D 拖入草稿、⌘1-4 整理 tag。

不做的

立意 sheet 不在主流程弹（暂停时才出）；划线不强制选 tag；倒计时不温和；思考引导不打断答题。
思考反馈分级：被动确认型（完成题统计 / 字数里程碑）默认 on，主动弹窗型（立意 prompt）默认 off，仅暂停时弹。

实现细节

稿纸：25×20 = 500 字 / 页，cell 约 26×28px，serif 17px。
输入法：候选词浮窗优先在格上方（避免遮挡写入区）。
中英 / 数字 / 标点：均强制 1 格（拟真考场习惯）。
平板软键盘弹出时稿纸压缩 + 上滚保持 cursor 可见。

数据 vs 视图

同一份草稿数据，两种 view：卡片视图（默认，扁平自由）+ 大纲视图（结构化按分论点分组）。切换不丢任何卡，只换组织方式。

大纲层级

立意（已写一句话）→ 3 个分论点 slot（用户自己命名，空 = "待补充"）→ 各分论点下嵌套素材卡 → 底部 未分配 区收纳无归属的卡。

交互

分论点头折叠 / 展开（chevron）。素材卡 拖动 handle 在分论点之间移动。"未分配"区允许自由打字 + 拖入素材。

AI 建议 · 克制

未分配卡 ≥1 张时显示 AI 建议条："…看起来属于 分论 N"。一键 采纳 自动归属，否则用户手动拖。不主动重排已分配卡。

分论点上限

默认 3 个分论点 slot（申论大作文标准三段结构）。需要更多再扩展（按"+ 添加分论点"）。立意写好 + 3 分论点 + 各论点下素材 = 完整答题大纲。

取舍：这个方案保住 v3 的自由度（卡片视图就是当前样子），同时给 reference 的大纲结构（切到大纲视图就是 reference 那种分类层级）。一键整理在两种 view 都能用，差别是大纲视图直接按"立意 → 分论 → 素材"输出结构化大纲，可一键导入答题。
