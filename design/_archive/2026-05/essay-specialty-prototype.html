申论 · 专项练习原型

跨卷单题答题模式. 用申论题库 745 套真题的 5 大类标签 (归纳概括 / 大作文 / 综合分析 / 公文 · 应用文 / 提出对策), 让用户选一类专攻, 在 2916 题里跨卷集中刷. 不限时, 用时计时器代替倒计时. 复用 ExamShell 的视觉与交互结构.

调性

备考同伴, 克制陪伴. 让备考从刷题变成思考. ink-first (黑为主), 蓝仅 focus / 链接 / 单关键 accent. 主 CTA 永远 --ink 底, 不 --accent. 不 italic 表强调, 不 emoji.

路由

列表 /essay/specialty · 答题 /essay/specialty/{questionId} · 评分结果 复用 /essay/grades/{recordId}.

分类

走 DB questions.canonical_subtype (master agent verify). 5 大类: 归纳概括 742 / 大作文 634 / 综合分析 609 / 公文 · 应用文 579 / 提出对策 352. 总 2916 题.

API

新加 GET /api/v2/essay/specialty/questions?subtype=&page=&pageSize= (paginate). 答题用现有 POST /api/v2/essay/grade (questionId + answerText). 评分跳现有 EssayGradingResult view.

不做

不创建 SimpleEssayExam 大改 — 单题答题复用 ExamShell, 通过 mode='single' prop 关掉 倒计时 / 题号 strip / Cmd-Enter 整卷交卷. 不动 essay V2 整卷模考路径 (并行不冲突).

实现要点 — 落地清单

Backend

新加 GET /api/v2/essay/specialty/questions?subtype=&page=&pageSize= 到 essay_v2.py. SQLAlchemy: WHERE question_kind='essay' AND canonical_subtype = :subtype + LIMIT N OFFSET M + LEFT JOIN essay_grading_records 取 latest createdAt (lastAnswered). Pydantic EssaySpecialtyQuestionListResponseV2: {items, total, page, pageSize}. pytest ≥ 3 case (5 类筛选 / 分页 / lastAnswered 标记).

Frontend

新 view EssaySpecialty.tsx + 复用 ExamShell with mode='single' prop 装 EssaySpecialtyExam.tsx. Router 加两条 path. Sidebar 申论 · 专项 从 disabled 改 to="/essay/specialty". apiQueries 加 fetchEssaySpecialtyQuestions + essaySpecialtyKeys.

ExamShell mode='single'

新增 prop mode?: 'paper' | 'single' (default 'paper' 不破老路径). single 时: 不渲染 QuestionStrip · 不渲染 5min warn · TopBar 隐藏 "暂停 / 整卷交卷" · 改 "提交评分" 单按钮 · Cmd-Enter 直接打开 SubmitDialog (一题简化版, 不带 D7=B 弃考拦截多题).
