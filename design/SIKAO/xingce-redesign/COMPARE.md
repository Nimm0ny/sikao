## 优劣分析

### A · classic-fenbi

**优点 (3 条)**
1. **学习成本零**: fenbi 用户开箱即用, 题号在左大字 + 圆形 letter 是数千万行测用户熟悉的视觉语法.
2. **scan 友好**: 单栏宽 720 + 上下题号 mono 标的清晰节奏让用户从 Q15 滑到 Q35 像翻文档, 不需要 "我现在在哪一题" 的额外 cognitive load.
3. **题目密度高**: 一屏 ~2-3 题, 适合考前刷量. 题组 (passage) 在 drawer 也用 paper-2 视觉聚集.

**缺点 (3 条)**
1. **brand 弱化**: 跟 fenbi 视觉太像, SIKAO 的 paper-warm + 暗朱 accent + serif 题干差异化感不强烈, 容易被误认作"fenbi 套皮".
2. **单栏浪费桌面**: 1920×1080 上 1200px 留白, 没有第二栏放笔记 / 进度.
3. **当前题指示弱**: 仅左侧 2px 朱线 + paper-2 渐隐, 长滚动后用户可能滚过头. 需要 keyboard shortcut [J] 跳到当前题. (本原型已实现 `focus-hint` 但没绑 J 键.)

### B · paper-quiet

**优点 (3 条)**
1. **brand 立得最稳**: serif 题干 + 章节式 chapter-label + 翻书节奏 + 双栏笔记位是 SIKAO "纸质阅读 / 留白" 调性最纯粹的表达, 跟 fenbi 划清界限.
2. **笔记一等公民**: 右栏 scratch-pad 常驻, 学行测真题时可以现场记 "差: 4,6,8,10 → 下一差=12 → 42" 这种 working trace, 答完不用切页就能复盘. 这是 fenbi / 公考通 / 粉笔都没有的差异化.
3. **降低焦虑**: 大留白 + 章节式 + 章末隐式 upsell (一行链接而非整张 upsell card) 让用户感觉 "在认真读书" 而不是 "在被催升级".

**缺点 (3 条)**
1. **桌面强依赖**: 双栏在 1024px 以下塌单栏, 笔记被挤到底部 sticky 失效. 移动端体验跟 A 几乎一样, 但样式工程量翻倍.
2. **节奏太慢**: chapter-label + s9=96 间距 + 单题 padding s7=48 → 一屏 ~1.5 题, 考前冲刺时用户嫌慢. 需要 Tweaks "compact" 模式压缩.
3. **方形 letter 学习成本**: 圆形是 fenbi 几乎所有公考用户都熟悉的, 方形会有~5% 用户报"不像答题".

### C · focus-rail

**优点 (3 条)**
1. **专注度最高**: 单题 focus + 上下题 dim 0.45 + opacity 0.7 hover 切, 完美避免 "刷到 Q35 才发现 Q3 没答" 的列表式陷阱. AI 时代用户更习惯单卡片切换 (ChatGPT / Anthropic 风格).
2. **AI 落点天然**: rail 常驻 + AI panel 在 expand 状态可见 + 选项 hover 红点提示陷阱 → Pro 用户画像 / 解题骨架 / 同类追加 都有"地方可放", 不挤压主区. 03b AI Pro 这个方案最贴合.
3. **键盘极致友好**: footer 的 keyboard hint + rail toggle (A 键) + 1-4 选项 + ←→ 切题 → 桌面 27" 用户可以全程不动鼠标, 跟 Linear / Superhuman 一类工具感强一致.

**缺点 (3 条)**
1. **mobile 几乎不可用**: focus 模式在 iPhone 上太挤, 上下题 dim 节省的认知负担在小屏上抵消不了空间损失. 已 ≤1024 塌堆叠但体验不如 A/B.
2. **题组 (03c) 适配差**: 题组共 passage 时 focus 模式只显当前题, passage 必须在 stem 上方贴, 但 stem 已经 19px serif 占空间. 需要 "passage 折叠/展开" 二次设计.
3. **accent 用量超预算**: 提交 CTA + topbar 1px progress + AI hint 红点 + Pro tag → 4 处 accent, 已超 style-guide "一屏最多 1-2 处". 真要落地必须砍 progress bar 或 Pro tag.

## 推荐方案 + 理由 (留给 master)

> subagent 不拍板. 列出每方案"特别值得保留点", 让 master 综合 / 可能融合.

### A 值得保留点
- **section-label "PART 01 / PART 02"** 的 mono eyebrow + serif chapter title 节奏 (任何方案都该用, B/C 已采纳)
- **题号 36px serif tabular**: 设计感最强, 仅 A 用. B/C 走 mono Q16 极简但 A 的视觉锚点更强 (题号是地标 / 不会迷路)
- **当前题左侧 2px 朱线** 比"整张卡 paper-2 高亮"克制, 是 SIKAO 暗朱 accent 的好用法

### B 值得保留点
- **右栏 scratch-pad 常驻**: SIKAO 跟 fenbi/粉笔最关键差异化. master 拍板时强烈建议保留这个右栏作为"learning trace 一等公民"模式的种子, 哪怕主方案选 A, 也可考虑 A + B 右栏融合
- **章节式 chapter-label** 翻书感: paper-warm 调性的高纯度表达
- **隐式 upsell** (一行链接 vs 整卡) 的克制态度: 比 A 的 upsell card 显得不功利

### C 值得保留点
- **rail AI panel 内嵌** 而非浮层: AI Pro 03b 最佳落点. 即便主方案不选 C, AI 解题骨架 / 用户画像 / AI 提问"该放在哪"这个问题, C 方案给了清晰答案
- **footer keyboard hint** 当 onboarding teaser: 让用户发现键盘可控, 重度用户会感谢
- **focus 模式上下题 dim**: B/C 都该考虑借用. A 改成"当前题外, 上下题 opacity 0.7" 也能立刻提升专注度

## 已知风险 / token 残留

- **0 hex/px 写死残留**: 全 3 文件审查过, 仅例外 `stroke-width="1.4"` (SVG 视觉细节, token 体系外, master 已许) + `box-shadow: 0 0 0 4px var(--accent-50)` (halo, 用 token color)
- **字号** 仍有 `17px` / `19px` 写死 — 这两个值不在 tokens.css `--t-*` 阶梯里 (阶梯只有 13/15/22/36/56/88), 是题干阅读字号 (`--read-fs lg/xl`), 跟 reading 切换 token (`[data-reading="lg"]`, `[data-reading="xl"]`) 协调; **可选改进**: 把题干 / 选项字号改用 `var(--read-fs)` + `var(--read-lh)` 让 Tweaks reading lg/xl 切换生效. 当前未做, 因为 spec 要求"答题区字号受 Tweak `reading` 控制" (`03-xingce.md` 验收点 1) → master 拍板时若选某方案, 实施时务必把题干/选项 `font-size: 17px / 19px` 改 `font-size: var(--read-fs)`, `line-height: var(--read-lh)`.
- **C 方案 accent 4 处** 超 1-2 处预算: master 选 C 时必砍 1-2 处 (建议砍 progress bar 或 Pro tag).
- **shadow halo** `box-shadow: 0 0 0 4px var(--accent-50)` 中 `4px` 是固定 halo 宽, 不在 token 间距阶梯; 跟 `s1=4px` 数值碰巧一致, 改 `var(--s1)` 可消除 px 残留 (3 处).
- **桌面 1024 以下** 的 C 方案体验已知差, 实施时建议 mobile 退回 A 列表式而非 C 折叠堆叠.
- **题组 03c 适配**: A/B 已天然兼容 (section + passage 上方), C 需二次设计 passage 折叠/锚点.
